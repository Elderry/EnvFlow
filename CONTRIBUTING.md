## ğŸ”€ Pull Request Process

1. **Fork** the repository and create your branch from `main`
2. **Code**: Follow the existing code style and conventions
3. **Test**: Ensure your changes work as expected
4. **Commit**: Write clear, concise commit messages
5. **Push**: Push your changes to your fork
6. **PR**: Create a pull request with a clear description of changes

### Code Style Guidelines

- Follow standard C# coding conventions
- Use meaningful variable and method names
- Comment complex logic
- Keep methods focused and concise
- Use async/await for asynchronous operations

### Commit Message Guidelines

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters
- Reference issues and pull requests when relevant
